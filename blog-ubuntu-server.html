Home Office Server Deployment and Automation for e-commerce

Hardware Project Report: Server Setup and Configuration

  • Burak Erdilli
  • Project Repository (Scripts & Read Me)
  • Detailed Documentation PDF

This project involved setting up a home server with automated deployment of microservices and custom Python scripts to manage an e-commerce platform on Trendyol. The server was optimized for performance and scalability, with automation scripts handling routine tasks such as inventory updates and product price adjustments. Additionally, the setup includes a personal cloud solution (Nextcloud), media streaming services, and local usage of large language models (LLMs).

Motivation

This project was driven by a deep belief in freedom and self-reliance. I’ve always been intrigued by the concept of self-hosting, where I have full control over my data without relying on external services. This aligns with my principle of digital sovereignty and avoiding vendor lock-in.

“I am a firm believer that if you give users freedom, they will solve their own problems in ways you never thought of.” – Linus Torvalds

In addition to the self-hosting aspect, I wanted to explore the complexities of microservices and the challenge of deploying secure, efficient setups on constrained hardware. My interest in hardware played a big role in this, as I was able to tinker with and repurpose older systems, pushing them to their limits while learning about networking, Dockerization, and resource optimization.

This project also served as a playground to deepen my understanding of modern technology stacks, while satisfying my curiosity and passion for creating efficient, resilient, and scalable systems.

Note: The hardware used is quite outdated and was specifically chosen to test and learn about services and configurations. This setup isn’t intended for heavy computational tasks but for exploring networking, resource management, and scaling on constrained systems.

In the future, a complete transition to dedicated hardware will be discussed, including detailed specifications and hardware selection.

Overview

This report documents the transformation of a Samsung Galaxy TabPro S and a Lenovo Ideapad Z570 into an Ubuntu-based media server and small cloud storage solution. The report covers hardware setup, software configurations, and potential use cases.

Device Specifications

Device #1: Lenovo Ideapad Z570

  • Processor: Intel(R) Core(TM) i5-2430M CPU @ 2.40GHz
  • RAM: 12GB
  • Storage: 256GB SSD
  • Ports: 3 x USB 2.0, 1 x USB 3.0, Ethernet (enp4s0): 10/100Mbps
  • Operating System: Ubuntu Server

Device #2: Samsung Galaxy TabPro S

  • Processor: Intel Core m3-6Y30
  • RAM: 4GB
  • Storage: 128GB SSD
  • Ports: Single USB Type-C Gen 3.1
  • Operating System: Ubuntu Server

Device #3 (Main Workstation)

  • Processor: AMD Ryzen 5 5600X, 6-Core, 12-Thread @ 3.7 GHz (Boost up to 4.6 GHz)
  • RAM: 32GB 3600 MHz
  • Storage: 5TB HDD, 1TB + 2TB NVMe SSD
  • Graphics: RTX 3090, 24GB GDDR6X
  • Operating System: Win10, Ubuntu
  • Ports:
    • USB 3.2 Gen 2 Type-C
    • USB 3.2 Gen 2 Type-A
    • USB 3.2 Gen 1 Type-A
    • USB 2.0
    • Gigabit Ethernet

Network topology illustrating the connections between the devices and their roles in the home office setup.

Hardware Setup

  • Adapters Used:
    • Type-C hub with USB and charging passthrough (Device #2 has a single Type-C port).
    • USB to Gigabit Ethernet adapter (Type-C port supports Gigabit speeds).
  • External Storage:
    • External SSD/HDD connected via Type-C hub for extended Nextcloud storage.
    • Future caching considerations to improve I/O performance.

Network Configuration

Ethernet Connection:
  • Device #2 uses a USB to Gigabit Ethernet adapter, though the bottleneck arises from the Type-C port’s 5Gbps limit.
  • Device #1’s old-generation 100 Mbit Ethernet port is limited in speed, used mainly for cron jobs and scripts.
  • A static IP was configured for consistent LAN access.
Wake-on-LAN (WOL):
  • Network adapter compatibility was verified for remote power management.
  • Wake-on-LAN was enabled for easy remote management.

Microservices Deployment

  • Nextcloud: Deployed for cloud storage with secure file sharing and remote access, configured in Docker containers for scalability.
  • Media Server: Set up for streaming media across devices, with metadata management and remote access.
  • Large Language Models: Deployed models like GPT for personal use, allowing real-time queries and tasks.
  • FastAPI App Testing: Tested RESTful services, request handling, authentication, and database management.

E-commerce Automation for Trendyol

  • Python Scripts for Automation: Developed to automate tasks like inventory management and price adjustments via Trendyol’s API. Scheduled tasks for ongoing updates.
  • Scripts include:
    • Price Adjustment: Automatically updates product prices based on criteria.
    • Sales Filtering: Adds high-view, high-favorite-rate products to Trendyol sales.
  • Over five weeks of automation, profits increased by 48.5%, views by 82.5%, and product variety grew by 24%.

Challenges & Solutions

  • SSH Connection Issue: Incorrect syntax. Solution: Corrected syntax and verified connection details.
  • SSH Authentication Failure: Authentication failed despite correct syntax. Solution: Enabled password authentication and reset password.
  • Docker Container Restart Loop: Misconfigured Docker setup. Solution: Fixed volume mounts and restarted containers.
  • Resource Constraints: Limited hardware resources caused performance issues. Solution: Optimized Docker containers and disabled unnecessary services.
  • LLM Web UI Issue: Port conflict preventing UI startup. Solution: Changed port and opened firewall.
  • Nextcloud Performance: Slow due to memory limits. Solution: Enabled Redis caching and increased PHP memory.
  • Nextcloud Installation: Incomplete setup. Solution: Re-ran installation and ensured dependencies were met.
  • Passwordless Boot: Server asked for a password on reboot. Solution: Configured passwordless reboot.
  • Preventing Sleep: Server suspended on inactivity. Solution: Disabled suspend and lid actions in settings.
  • Static IP Configuration: Dynamic IP issue. Solution: Configured static IP via Netplan.
  • Wake-on-LAN Issue: Unreliable WoL. Solution: Enabled and persisted WoL settings.
  • Charging Monitoring: Adapter limitations affected power. Solution: Replaced with a more capable hub for improved power.
